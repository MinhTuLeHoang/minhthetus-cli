# Configuration Directory (`~/.minhthetus-cli/`)

The `~/.minhthetus-cli/` directory serves as the local configuration and support hub for the `minhthetus-cli`. It stores persistent data, shell integration scripts, and autocompletion logic to provide a seamless experience across different terminal sessions.

## Directory Overview

This hidden directory is automatically created in your home folder when you first run certain `minhthetus-cli` commands or during the initial setup.

## Files and Their Purposes

### 1. `git-accounts.json`
- **File Type**: `JSON`
- **Data Interface**:
  ```typescript
  interface GitAccount {
    title: string; // Display name for the account (e.g., "Work", "Personal")
    name: string;  // Git user.name
    email: string; // Git user.email
  }

  type data = GitAccount[];
  ```
- **Purpose**: Stores a list of Git identities (titles, names, and emails).
- **Supported CLI**: `minhthetus-cli git account`
- **Functionality**: 
  - Allows you to save multiple Git profiles (e.g., Work and Personal).
  - Enables quick switching of Git identity for the current repository.
  - Managed via the `--manage` flag: `minhthetus-cli git account --manage`.

### 2. `list-repo.json`
- **File Type**: `JSON`
- **Data Interface**:
  ```typescript
  interface RepoConfig {
    name: string;        // Name of the repository
    description: string; // Description from package.json
    path: string;        // Absolute path to the repository
  }

  type data = RepoConfig[];
  ```
- **Purpose**: Tracks local repositories for bulk operations.
- **Supported CLI**: `minhthetus-cli web check-malware`, `minhthetus-cli git list-repo`, `minhthetus-cli repo-untrack <path>`
- **Functionality**:
  - Stores metadata and paths for projects registered with the CLI.
  - Used for scanning multiple local projects for vulnerable or malicious packages.
  - Interactive management via the `git list-repo` command.
  - Automatically cleaned up if paths are found to be invalid during a scan.

### 3. `shell-wrapper.sh`
- **File Type**: `Shell Script`
- **Interface/Behavior**:
  - Modifies the current shell session by sourcing a temporary file generated by the CLI.
  - Intercepts calls to the `minhthetus-cli` command.
  - Environment Dependency: Uses `MINHTHETUS_SHELL_PIPE`.
- **Purpose**: A shell function wrapper that intercepts `minhthetus-cli` calls.
- **Supported CLI**: All commands, but specifically enables environment-modifying commands like `web start` or scripts that use `nvm use`.
- **Functionality**:
  - The CLI normally runs in a subshell, meaning it cannot change the environment of your primary terminal (like changing directories or switching Node versions).
  - This wrapper captures specific shell commands emitted by the CLI into a temporary file and "sources" them in your current session.
  - Automatically added to your shell profile (`.zshrc` or `.bash_profile`) during setup.

### 3. `completion.sh`
- **File Type**: `Shell Script`
- **Purpose**: Provides tab-based autocompletion for `minhthetus-cli` commands and options.
- **Supported CLI**: Global `minhthetus-cli` command.
- **Functionality**:
  - Contains the shell logic required for your terminal (Zsh, Bash) to suggest subcommands and flags.
  - It is a static script generated by the CLI to ensure autocompletion is fast and responsive.
  - Sourced in your shell profile during the setup process.

---

## Technical Details

- **Creation**: The directory and its scripts are primarily managed by internal utility modules:
  - `src/lib/utils/setup-shell-wrapper.js`: For the shell wrapper.
  - `src/lib/setup-completion.js`: For the autocompletion script.
  - `src/scripts/git/account.sh`: For the Git identity storage.
- **Permissions**: Configuration files like `git-accounts.json` are often set with restricted permissions to protect your identity information.
